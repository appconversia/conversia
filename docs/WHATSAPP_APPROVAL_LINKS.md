# Enlaces para aprobación de la aplicación WhatsApp API en Meta

Usa estos enlaces al enviar tu aplicación a revisión (App Review) de Meta para WhatsApp Business API.

## Envío a revisión (App Review)

- **App Review – Inicio**  
  https://developers.facebook.com/docs/apps/review  

- **Antes de enviar (requisitos)**  
  https://developers.facebook.com/docs/resp-plat-initiatives/app-review/before-you-submit  

- **Envío de muestra – WhatsApp Business Platform**  
  https://developers.facebook.com/docs/whatsapp/embedded-signup/app-review/sample-submission/  

- **Permisos de WhatsApp (qué solicitar)**  
  https://developers.facebook.com/docs/whatsapp/permissions  

  Permisos típicos para negocio:
  - `whatsapp_business_messaging` – envío y recepción de mensajes.
  - `whatsapp_business_management` – gestión de números y plantillas.

- **Policy Enforcement (WhatsApp)**  
  https://developers.facebook.com/docs/whatsapp/overview/policy-enforcement/

## Requisitos típicos antes de enviar

1. App en producción y accesible (ej. https://chat.yjbarriles.com).
2. Icono de la app 1024x1024 según especificaciones de Meta.
3. Al menos una llamada correcta a la API por cada permiso solicitado en los últimos 30 días.
4. Vídeos en alta resolución (1080p o superior) mostrando el uso de cada permiso.
5. Descripción clara de cómo se usa cada permiso y enlace a políticas (privacidad, términos, eliminación de datos) con las URLs de arriba.

Verificación de cumplimiento del proyecto: ver [META_WHATSAPP_POLICY_COMPLIANCE.md](./META_WHATSAPP_POLICY_COMPLIANCE.md).
